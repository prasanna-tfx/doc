01 First Step

Connection & Authentication

Connect via WebSocket to the backend proxy endpoint. Authentication happens at the HTTP upgrade stage using a JWT passed as a query parameter (recommended for mobile) or via cookie.

Endpoint

textWebSocket URL
wss://your-backend.com/api/doc-brain/realtime-ws

// Recommended: JWT as query param
wss://your-backend.com/api/doc-brain/realtime-ws?token=<JWT>

Flutter Connection

dartConnecting with web_socket_channel
import 'dart:convert';
import 'package:web_socket_channel/web_socket_channel.dart';

class RealtimeSessionService {
  WebSocketChannel? _channel;

  void connect(String jwt) {
    final uri = Uri.parse(
      'wss://your-backend.com/api/doc-brain/realtime-ws?token=$jwt',
    );

    _channel = WebSocketChannel.connect(uri);

    _channel!.stream.listen(
      (message) => _handleEvent(jsonDecode(message as String)),
      onError: (error) => _handleError(error),
      onDone: () => _handleDisconnected(),
    );
  }

  void _handleEvent(Map<String, dynamic> event) {
    final type = event['type'] as String;
    switch (type) {
      case 'session.ready':       _onSessionReady(event);     break;
      case 'session.interrupted': _onInterrupted();           break;
      case 'session.end_requested': _onSessionEnd(event);    break;
      case 'tool.running':        _onToolRunning(event);      break;
      case 'tool.done':           _onToolDone(event);         break;
      case 'tool.aborted':        _onToolAborted(event);      break;
      case 'response.audio.delta': _onAudioDelta(event);     break;
      case 'response.audio.done': _onAudioDone();             break;
      case 'response.audio_transcript.done': _onTranscript(event); break;
      case 'response.done':       _onResponseDone(event);    break;
      case 'error':               _onError(event);            break;
    }
  }
}
⚠️
Rate Limits (enforced server-side) Max 2 concurrent sessions per user and 10 connections per minute. Exceeding either closes the socket with code 4029. Show an error and prevent any reconnect loop.
02 Session Lifecycle

Session Ready & State Machine

After the WebSocket upgrades, the backend configures OpenAI and emits session.ready. Only start streaming microphone audio after this event. Mirror the session phase in your UI state.

Session Phases

idle
No active conversation turn.
UI: Show mic button / ready indicator
user_speaking
VAD detected voice — user is speaking.
UI: Show waveform animation / "Listening…"
ai_responding
AI is generating a reply and streaming audio.
UI: Show AI speaking animation, stream audio playback
tool_executing
Backend is running a tool call (doc lookup, search, etc).
UI: Show spinner / "Looking up…" status from tool.running
interrupted
User spoke while AI was responding — AI was cut off.
UI: Stop playback immediately, return to listening
💡
Phase is informational The backend tracks the authoritative phase. The app doesn't receive phase values directly — derive it from the events you receive (response.created → ai_responding, tool.running → tool_executing, etc).
03 Microphone

Mic Audio Streaming

Continuously stream mic audio as base64-encoded PCM16 chunks after receiving session.ready. The server VAD automatically detects speech start/end — the app does not manage turns.

Event to Send

jsoninput_audio_buffer.append — sent every 20–50ms
{

Flutter Implementation

dartMic capture using record package
import 'dart:convert';
import 'dart:typed_data';
import 'package:record/record.dart';

class MicStreamService {
  final AudioRecorder _recorder = AudioRecorder();

  Future<void> startStreaming(WebSocketChannel channel) async {
    // Request microphone permission first
    final hasPermission = await _recorder.hasPermission();
    if (!hasPermission) throw Exception('Microphone permission denied');

    final stream = await _recorder.startStream(const RecordConfig(
      encoder: AudioEncoder.pcm16bits,  // Raw PCM16
      sampleRate: 16000,                // 16 kHz
      numChannels: 1,                   // Mono
    ));

    stream.listen((Uint8List chunk) {
      // chunk is already raw PCM16 little-endian bytes
      final b64Audio = base64Encode(chunk);
      channel.sink.add(jsonEncode({
        'type': 'input_audio_buffer.append',
        'audio': b64Audio,
      }));
    });
  }

  Future<void> stopStreaming() async {
    await _recorder.stop();
  }
}
🚫
Never send these events manually Do NOT send input_audio_buffer.commit, response.create, or session.update. The backend VAD manages all of this. Sending them will cause double-responses or corrupt session state.
04 Speaker

Audio Playback

The server forwards OpenAI's response.audio.delta events as raw forwarded JSON. Decode each chunk and feed it to an audio queue. Stop immediately on interruption.

Key Events

OpenAI fwd response.audio.delta
⚡ Append to playback buffer immediately
Streamed PCM16 audio chunk for the current AI response.
jsonResponse shape
{
  "type": "response.audio.delta",
  "response_id": "resp_abc123",
  "item_id": "item_xyz",
  "output_index": 0,
  "content_index": 0,
  "delta": "UklGRiQAAABXQVZFZm10IBAAAA..."  // base64 PCM16 at 24kHz mono
}
OpenAI fwd response.audio.done
✅ Flush audio queue
All audio chunks for this response turn have been sent.
json
{
  "type": "response.audio.done",
  "response_id": "resp_abc123",
  "item_id": "item_xyz",
  "output_index": 0,
  "content_index": 0
}

Flutter Playback

dartPCM16 streaming playback
import 'dart:async';
import 'dart:convert';
import 'dart:typed_data';
import 'package:just_audio/just_audio.dart';

class PcmPlaybackService {
  final AudioPlayer _player = AudioPlayer();
  final _buffer = StreamController<Uint8List>.broadcast();
  bool _isPlaying = false;

  Future<void> init() async {
    // Set audio source to a custom PCM16 stream at 24kHz
    await _player.setAudioSource(
      _Pcm16StreamSource(stream: _buffer.stream),
    );
  }

  void appendChunk(String base64Delta) {
    final bytes = base64Decode(base64Delta);
    _buffer.add(bytes);
    if (!_isPlaying) {
      _player.play();
      _isPlaying = true;
    }
  }

  // Call this on session.interrupted — critical!
  void stopAndClear() {
    _player.stop();
    _isPlaying = false;
    // Drain pending chunks
  }

  void dispose() {
    _buffer.close();
    _player.dispose();
  }
}

// In your event handler:
void _onAudioDelta(Map<String, dynamic> event) {
  playback.appendChunk(event['delta'] as String);
}

void _onInterrupted() {
  playback.stopAndClear(); // ← MUST stop immediately
}

dartSending image from Flutter (with compression)
import 'package:image_picker/image_picker.dart';
import 'package:flutter_image_compress/flutter_image_compress.dart';

Future<void> sendImage({String? caption}) async {
  final picker = ImagePicker();
  final file = await picker.pickImage(source: ImageSource.gallery);
  if (file == null) return;

  // Compress to stay under ~4 MB base64 limit
  final compressed = await FlutterImageCompress.compressWithFile(
    file.path,
    quality: 80,
    minWidth: 1280,
    minHeight: 1280,
  );
  if (compressed == null) return;

  final b64 = base64Encode(compressed);
  final dataUrl = 'data:image/jpeg;base64,$b64';

  final content = [
    {'type': 'input_image', 'image_url': dataUrl, 'detail': 'auto'},
    if (caption != null) {'type': 'input_text', 'text': caption},
  ];

  _channel.sink.add(jsonEncode({
    'type': 'conversation.item.create',
    'item': {'type': 'message', 'role': 'user', 'content': content},
  }));
}

// Handle image validation errors from server:
void _onError(Map<String, dynamic> event) {
  final errType = event['error']?['type'];
  if (errType == 'image_validation_error') {
    showSnackBar(event['error']['message']);
  }
}

07 Interruption

Interruption Handling

When the user speaks while the AI is responding, the server cancels the AI response and aborts in-flight tools automatically. The app's only job is to stop audio playback the moment session.interrupted arrives.

Interruption Flow

1 Server (VAD) Detects speech on input audio buffer
2 Server → App session.interrupted sent to Flutter immediately
3 App ← action Stop audio playback and clear buffer NOW. This is critical — failure causes overlapping voices.
4 Server → OpenAI Sends response.cancel to OpenAI with the active response ID
5 Server → App tool.aborted for any in-flight tool calls
6 Server → App response.cancelled forwarded from OpenAI
7 Server (VAD) Detects speech stop, sends response.create to OpenAI automatically
8 Server → App New AI response begins — response.audio.delta resumes

Key Events

Custom session.interrupted
🛑 Stop audio playback NOW
Fired the instant VAD detects user speech during AI response.
json
{
  "type": "session.interrupted"
}
OpenAI fwd response.cancelled
Discard any buffered audio chunks
Confirms the AI response was cancelled. Discard any remaining buffered audio.
json
{
  "type": "response.cancelled",
  "response_id": "resp_abc123"
}
08 Lifecycle

Session End & Cleanup

Sessions end via AI request, user action, or network events. Always clean up mic and audio resources.

Custom session.end_requested
Show completion UI, close gracefully
The AI determined the session is complete. An optional reason is provided.
json
{
  "type": "session.end_requested",
  "reason": "Consultation complete. Have a great day!"  // nullable
}

Close Codes

Close CodeReasonApp Action
4029Rate limit exceeded (sessions or connections/min)Show error, wait 60s, offer retry
1011Server errorExponential backoff: 2s → 4s → 8s → max 30s
1000Normal closureClean shutdown, no reconnect needed
no code / 1006Network dropAuto-reconnect with backoff, restore state
dartCleanup + reconnect logic
int _reconnectAttempts = 0;

Future<void> endSession() async {
  await mic.stopStreaming();
  playback.stopAndClear();
  await _channel?.sink.close();
}

void _handleDisconnected() {
  final code = _channel?.closeCode;
  switch (code) {
    case 4029:
      showError('Too many active sessions. Please wait 60 seconds.');
      return; // Don't auto-reconnect
    case 1000:
      return; // Normal close, no reconnect
    default:
      _scheduleReconnect();
  }
}

void _scheduleReconnect() {
  if (_reconnectAttempts >= 5) {
    showError('Connection failed after 5 attempts.');
    return;
  }
  final delay = Duration(seconds: min(30, pow(2, _reconnectAttempts).toInt()));
  Future.delayed(delay, connect);
  _reconnectAttempts++;
}

void _onSessionReady(Map<String, dynamic> event) {
  _reconnectAttempts = 0; // Reset on successful connection
  mic.startStreaming(_channel!);
}
